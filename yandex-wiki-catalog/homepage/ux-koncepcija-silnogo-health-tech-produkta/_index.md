## Основная идея

Самая большая ошибка медицинских приложений:

```
они показывают данные
```

Пользователь хочет:

```
понять проблему
```

---

## Основная идея UX

Продукт строится вокруг:

```
ПРОБЛЕМА
```

Не вокруг:

```
анализов
```

---

## Основная модель интерфейса

```
Problem → Hypotheses → Events → Outcomes
```

---

## Главный экран

```
Мои проблемы
```

Пример:

```
• Мигрени
• Усталость
• Боль в колене
```

---

## Экран проблемы

Это главный экран системы.

```
Проблема: Усталость
```

Вкладки:

```
Timeline
Hypotheses
Symptoms
Analytics
```

---

## Timeline

Хронология.

```
15 марта — анализ крови
10 марта — консультация
5 марта — начало лечения
```

Каждое событие показывает:

```
результат
резолюцию
комментарий
```

---

## Hypotheses

Гипотезы причин.

Пример:

```
дефицит железа
гипотиреоз
хронический стресс
```

Статусы:

```
testing
confirmed
rejected
```

---

## Analytics

AI показывает:

```
инсайты
паттерны
риски
```

Пример:

```
Ваш ферритин падает последние 6 месяцев.
```

---

## Симптомы

Пользователь добавляет:

```
усталость
головокружение
слабость
```

AI связывает симптомы и анализы.

---

## Экран анализа

```
таблица показателей
AI интерпретация
графики
```

Пример:

```
Ферритин — 12 (ниже нормы)
```

---

## Экран динамики

График.

```
X — дата
Y — значение
```

Показывает:

```
референс
норму
тренд
```

---

## AI Insights

AI показывает:

```
возможные причины
следующие шаги
```

---

## Экран лечения

```
курс лечения
```

Содержит:

```
препараты
дозировки
график
результат
```

---

## Самая важная UX функция

**Health Timeline**

Пользователь видит:

```
симптом
↓
анализ
↓
диагноз
↓
лечение
↓
результат
```

---

## Вторая killer-функция

**AI Second Opinion**

AI анализирует:

```
всю историю
```

---

## Третья killer-функция

**Medical Pattern Detection**

AI находит:

```
повторяющиеся проблемы
скрытые связи
```

Пример:

```
мигрень возникает после плохого сна
```

---

## UX-метрика успеха

Пользователь должен ответить на вопрос:

```
Почему мне плохо?
```

Если приложение отвечает на этот вопрос — продукт работает.
