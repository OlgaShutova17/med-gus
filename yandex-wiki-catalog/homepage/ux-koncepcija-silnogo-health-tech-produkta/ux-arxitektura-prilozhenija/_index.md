## UX-архитектура приложения

Очень важно:

**Главная модель UX — timeline лечения.**

Пользователь должен видеть **историю решения проблемы**.

---

## Основные разделы

```
Dashboard
Themes
Events
Analyses
Research
Consultations
AI assistant
Profile
```

---

## Главный экран

```
Главная
```

Отображает:

```
активные проблемы
последние события
AI рекомендации
```

Пример:

```
Мои проблемы

• Мигрени
• Боль в колене
• Повышенный сахар
```

---

## Экран проблемы

```
Theme page
```

Основной экран продукта.

Структура:

```
Название проблемы
Описание
```

Далее вкладки:

```
Timeline
Hypotheses
Symptoms
Analytics
```

---

## Timeline

Хронологическая лента.

```
12.03 анализ крови
10.03 прием терапевта
05.03 МРТ
```

Карточка события:

```
тип события
дата
результат
резолюция
```

---

## Экран анализа

```
Analysis page
```

Отображает:

```
таблица показателей
AI интерпретация
файлы
```

Пример:

```
Гемоглобин — 140
Ферритин — 12 (ниже нормы)
```

---

## Экран динамики

График показателей.

```
X — дата
Y — значение
```

Отображается:

```
референс диапазон
норма
отклонения
```

---

## Экран гипотез

```
Hypotheses
```

Пример:

```
1. Дефицит железа
2. Гастрит
3. Хронический стресс
```

Статусы:

```
testing
confirmed
rejected
```

## Экран курса лечения

```
Treatment course
```

Содержит:

```
препараты

прогресс выполнения
```

## AI Assistant

Чат с AI.

Пользователь пишет:

```
Почему высокий сахар?
```

AI отвечает:

```
Возможные причины:

[tab:инсулинорезистентность::]
[/tab]
[tab:питание::]
[/tab]
[tab:стресс::]
```

---

## Экран добавления события

```
Add event
```

Пользователь выбирает:

```
анализ
консультация
исследование
другое
```

---

## Добавление анализа

Два режима:

```
Загрузить файл
Ввести вручную
```

После OCR пользователь **подтверждает данные**.

---

## Экран резолюции

После события пользователь выбирает:

```
Помогло
Не помогло
```

И добавляет комментарий.

---

## Полный пользовательский путь

```
регистрация
   ↓
создать проблему
   ↓
добавить симптомы
   ↓
добавить гипотезу
   ↓
загрузить анализ
   ↓
получить AI интерпретацию
   ↓
добавить консультацию
   ↓
создать курс лечения
   ↓
оценить результат
```

---

## Самая важная UX-идея

Продукт должен отвечать на вопрос:

```
ПОЧЕМУ мне плохо
```

Поэтому интерфейс строится вокруг:

```
Проблема
↓
Гипотезы
↓
События
↓
Результаты
```
