Ты задал **очень правильный вопрос**:

> как объединить
> **гипотезы — события — выводы**
> и **хронологическую историю**

Это центральная проблема продукта.

---

## Проблема обычного timeline

Классический timeline:

```
1 марта — анализ
10 марта — врач
20 марта — лечение
```

Но пользователь **не понимает смысл**.

---

## Модель мышления человека

Пользователь думает так:

```
мне плохо
↓
почему?
↓
что проверить?
↓
что помогло?
```

---

## Поэтому нужна двойная модель интерфейса

## 1️⃣ Хронология (timeline)

понятна пользователю

## 2️⃣ Гипотезы (diagnostic tree)

понятна для анализа

---

## UX решение — dual view

Экран проблемы:

```
Проблема: Хроническая усталость
```

Две вкладки:

```
Timeline
Hypotheses
```

---

## Timeline

Привычная история:

```
10 янв — симптомы
15 янв — анализ крови
20 янв — консультация
25 янв — лечение
```

---

## Hypothesis view

Это **исследование проблемы**.

```
Возможные причины:

• дефицит железа
• гипотиреоз
• хронический стресс
```

---

## Связь гипотез и событий

Каждое событие влияет на гипотезу.

Пример:

```
анализ ферритина
↓
поддерживает гипотезу "дефицит железа"
```

---

## UI карточка события

```
15 марта
Анализ крови

ферритин 10 (ниже нормы)

влияние:

✓ подтверждает гипотезу "дефицит железа"
```

---

## Визуальная модель

Можно показать:

```
Hypothesis
     ↓
 Events
     ↓
 Evidence
```

---

## Пример

```
Гипотеза:
Дефицит железа

Подтверждения:

✓ ферритин низкий
✓ гемоглобин снижается

Опровержения:

✗ нормальный B12
```

---

## Итог для пользователя

Вместо:

```
много анализов
```

он видит:

```
наиболее вероятная причина
```

## Почему это очень мощно

Обычные медицинские приложения показывают:

```
данные
```

Эта система показывает:

```
расследование
```

---

## Очень важная идея

Система становится:

```
medical investigation engine
```

## Проблема

Есть две модели:

### Timeline

```
10 марта — анализ
15 марта — врач
20 марта — лечение
```

### Hypothesis model

```
гипотеза
↓
доказательства
↓
результат
```

Обычно приложения показывают **только timeline**.

Это плохо.

---

## Как думает пользователь

Пользователь думает так:

↓

↓

```

## Поэтому нужна двойная модель

```

## Investigation View

Это **граф расследования**.

```
Hypothesis

дефицит железа
```

```
Evidence
```

```
✓ ферритин 10
✓ гемоглобин 120
✓ усталость
```

---

## Визуальная модель

```
           Symptoms
              │
              ▼
         Hypothesis
              │
      ┌───────┼───────┐
      ▼               ▼
   Analysis        Research
      │               │
      ▼               ▼
   Evidence        Evidence
```

---

## Связь событий с гипотезами

Каждое событие влияет на гипотезу.

```
analysis → supports hypothesis
```

или

```
analysis → rejects hypothesis
```
