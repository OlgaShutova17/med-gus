---
title: Выбор резолюции события
---

## User Story

Как пользователь, я хочу отмечать резолюцию «Помогло» / «Не помогло» для событий, чтобы отслеживать эффективность проводимых манипуляций и корректировать план лечения.

## Приоритет

Средний

## Предусловия

- Пользователь авторизован в системе
- Пользователь находится в разделе «События»
- Функция «Резолюции» активна для пользователя

## Основной сценарий

1. Пользователь выбирает событие
2. Событие выделяется и становится активным
3. Пользователь выбирает резолюцию: «Помогло» или «Не помогло»
4. При необходимости добавляет комментарий с деталями
5. Нажимает «Сохранить»
6. Резолюция сохраняется и отображается в интерфейсе

## Альтернативные сценарии

### A1. Массовая маркировка нескольких событий

1. Пользователь выбирает несколько событий
2. Выставляет резолюции для каждого
3. Нажимает «Сохранить»
4. Все выбранные резолюции сохраняются

### A2. Редактирование ранее выставленной резолюции

1. Пользователь выбирает событие с уже выставленной резолюцией
2. Меняет значение (например, с «Помогло» на «Не помогло»)
3. При необходимости обновляет комментарий
4. Нажимает «Сохранить»
5. Обновлённая резолюция сохраняется в базе данных

## Критерии приёмки

- [ ] Пользователь может выставить резолюцию «Помогло» или «Не помогло» для любого события
- [ ] Доступно поле для комментария с деталями резолюции
- [ ] Возможна массовая маркировка нескольких событий за один раз
- [ ] Ранее выставленную резолюцию можно изменить
- [ ] Результаты сохраняются и отображаются в интерфейсе

## GWT-сценарии

### Выставление резолюции

**Given** авторизованный пользователь находится в разделе «События»
**When** он выбирает событие, отмечает резолюцию и нажимает «Сохранить»
**Then** система сохраняет резолюцию и отображает её в интерфейсе рядом с событием

### Редактирование резолюции

**Given** событие с ранее выставленной резолюцией «Помогло»
**When** пользователь меняет резолюцию на «Не помогло» и нажимает «Сохранить»
**Then** система обновляет резолюцию события в базе данных
