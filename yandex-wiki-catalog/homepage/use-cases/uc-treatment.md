## User Story

Как пациент, я хочу создавать и отслеживать курсы лечения, чтобы контролировать выполнение назначений врача и соблюдать график терапии.

## Приоритет

Высокий

## Предусловия

- Пользователь авторизован в системе
- Внутри тематики создана хотя бы одна гипотеза (курс лечения привязывается к гипотезе)

---

## UC-1: Создание курса лечения

### Основной сценарий

1. Пользователь нажимает «Новый курс лечения» внутри гипотезы
2. Заполняет форму: название, врач, дата начала, диагноз
3. Созданный курс отображается в разделе «Курсы лечения»
4. Пользователь добавляет лекарственные препараты: название, дозировка, частота приёма
5. При необходимости прикрепляет фото рецепта
6. Система сохраняет курс и отображает его с кнопкой «Редактировать»

### Альтернативные сценарии

#### A1. Редактирование курса

1. Пользователь нажимает «Редактировать»
2. Система открывает редактор курса
3. Пользователь вносит изменения и нажимает «Сохранить изменения»
4. Система сохраняет изменения и отображает обновлённые данные

### Критерии приёмки

- [ ] Форма содержит поля: название, врач, дата начала, диагноз
- [ ] Внутри курса можно добавлять препараты с дозировкой и частотой приёма
- [ ] Поддерживается загрузка фото/скриншота рецепта
- [ ] Доступна кнопка «Редактировать» для изменения данных курса

---

## UC-2: Отслеживание курса лечения

### Основной сценарий

1. Система отправляет PUSH-уведомление с напоминанием о следующем приёме
2. Пользователь отмечает «Пройдено» / «Не пройдено»
3. Система принимает отметку и предлагает внести данные с результатами этапа (текстовый комментарий, изображение документа)

### Критерии приёмки

- [ ] Поля для отметок «пройдено» / «не пройдено» присутствуют
- [ ] Доступно поле для описания результатов по итогу курса
- [ ] Отображается процент прохождения курса (опционально)
