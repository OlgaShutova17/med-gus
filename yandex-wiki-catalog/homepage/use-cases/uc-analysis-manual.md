title: Добавление анализа ручным вводом
---

## User Story

Как пользователь, я хочу вручную вводить результаты анализов, чтобы фиксировать их даже в случаях, когда файл недоступен (бумажный бланк, портал без выгрузки).

## Приоритет

Высокий

## Предусловия

- Пользователь зарегистрирован и авторизован в системе
- У пользователя есть результат анализа, который он может прочитать, но не имеет файла для прикрепления

## Бизнес-правила

1. Доступ только авторизованному пользователю
2. Анализ сохраняется только при наличии: типа анализа, даты проведения и хотя бы одного заполненного поля результата
3. Тип анализа выбирается из преднастроенного списка (4000+ типов) или «другое»
4. В зависимости от типа анализа доступен структурированный ввод (название показателя, значение, единицы, референсные диапазоны) или общее текстовое поле
5. Числовые поля принимают только целые или дробные числа; булевые — да/нет
6. Каждый анализ привязан к профилю пользователя

## Основной сценарий

1. Пользователь переходит в раздел «Анализы»
2. Нажимает кнопку «Добавить анализ вручную»
3. Система отображает форму: тип анализа, дата проведения, поля результатов
4. Пользователь выбирает тип анализа из списка
5. Указывает дату проведения
6. Вводит данные результата (значения показателей или текстовое заключение)
7. Нажимает «Сохранить»
8. Система валидирует обязательные поля
9. Система сохраняет запись и привязывает к профилю пользователя
10. Анализ отображается в общем списке

## Альтернативные сценарии

### A1. Не заполнены обязательные поля

1. Пользователь нажимает «Сохранить» без типа, даты или данных результата
2. Система подсвечивает проблемные поля («Выберите тип анализа», «Укажите дату», «Введите хотя бы один результат»)
3. Запись не создаётся, форма остаётся открытой

## Критерии приёмки

- [ ] Доступен режим ручного ввода анализа
- [ ] Тип анализа выбирается из списка
- [ ] Поддерживается структурированный ввод показателей с единицами и референсами
- [ ] При сохранении с обязательными полями запись создаётся и отображается в списке
- [ ] Данные доступны после повторного входа
- [ ] При попытке сохранения без обязательных данных отображается ошибка

## GWT-сценарии

### Успешное добавление анализа вручную

**Given** авторизованный пользователь находится в режиме ручного ввода анализа
**When** он выбирает тип, указывает дату и вводит результаты, затем нажимает «Сохранить»
**Then** система создаёт запись и показывает её в общем списке анализов

### Валидация обязательных полей

**Given** авторизованный пользователь находится в режиме ручного ввода анализа
**When** он нажимает «Сохранить», не указав тип, дату или результаты
**Then** система не создаёт запись и показывает сообщение о необходимости заполнить обязательные поля
