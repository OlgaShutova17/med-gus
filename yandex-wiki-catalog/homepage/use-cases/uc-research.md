title: Добавление результата исследования
---

## User Story

Как пользователь, я хочу добавлять результаты инструментальных исследований (МРТ, КТ, УЗИ, ЭКГ), чтобы хранить заключения и снимки из разных клиник в одном месте.

## Приоритет

Высокий

## Предусловия

- Пользователь зарегистрирован и авторизован в системе
- У пользователя есть результат исследования (заключение, протокол, снимки)

## Бизнес-правила

1. Доступ только авторизованному пользователю
2. Исследование сохраняется только при наличии: типа исследования, даты проведения и хотя бы одного поля результата или прикреплённого файла
3. Тип исследования выбирается из преднастроенного списка (МРТ, КТ, УЗИ, ЭКГ и т.д. с уточнениями, например «МРТ головного мозга»); при необходимости — «другое»
4. Дата проведения не может быть позже текущей даты
5. Допустимые форматы файлов: PDF, PNG, JPG; максимальный размер — 25 МБ
6. Каждое исследование привязано к профилю пользователя и не отображается другим пользователям
7. В структурированных полях вводятся только корректные значения по типу поля; в текстовых — свободный текст

## Основной сценарий

1. Пользователь переходит в раздел «Исследования»
2. Нажимает «Добавить исследование»
3. Система отображает форму: тип исследования, дата проведения, поля результата/заключения, поле для файлов
4. Пользователь выбирает тип исследования
5. Указывает дату проведения
6. Вводит данные результата (заключение, ключевые находки, рекомендации) и при необходимости прикрепляет файл
7. Нажимает «Сохранить»
8. Система валидирует обязательные поля, формат и размер файла
9. Система сохраняет запись и привязывает к профилю пользователя
10. Отображается уведомление об успехе, исследование появляется в списке

## Альтернативные сценарии

### A1. Не заполнены обязательные поля

1. Пользователь нажимает «Сохранить» без типа, даты или данных результата
2. Система подсвечивает проблемные поля: «Выберите тип исследования», «Укажите дату», «Введите хотя бы один результат»
3. Запись не создаётся, форма остаётся открытой

### A2. Неподдерживаемый формат или превышен размер файла

1. Пользователь выбирает файл недопустимого формата или размером >25 МБ
2. Система отклоняет файл и сообщает о допустимых форматах и ограничении размера

## Критерии приёмки

- [ ] Авторизованный пользователь может открыть раздел «Исследования» и форму добавления
- [ ] Тип исследования выбирается из списка (включая «другое»)
- [ ] Система не допускает дату позже текущей
- [ ] Доступен структурированный ввод и свободный текст заключения
- [ ] Прикрепление файла допустимого формата и размера работает корректно
- [ ] При заполненных обязательных полях запись создаётся и отображается в списке
- [ ] После повторного входа исследования доступны в списке
- [ ] При отсутствии обязательных данных отображается сообщение об ошибке

## GWT-сценарии

### Успешное добавление исследования

**Given** авторизованный пользователь находится на экране добавления исследования
**When** он выбирает тип, указывает дату, вводит результаты и нажимает «Сохранить»
**Then** система создаёт запись и показывает её в общем списке исследований

### Валидация обязательных полей

**Given** авторизованный пользователь находится на экране добавления исследования
**When** он нажимает «Сохранить» без типа, даты или данных результата
**Then** система не создаёт запись и показывает сообщение о необходимости заполнить обязательные поля

### Доступность после повторного входа

**Given** у пользователя ранее сохранены исследования
**When** он выходит из аккаунта и снова входит
**Then** система отображает ранее сохранённые исследования в общем списке
