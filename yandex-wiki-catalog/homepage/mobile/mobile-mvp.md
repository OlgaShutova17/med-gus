## Зачем мобайл

Веб решает задачу «разобраться дома». Мобайл решает задачу **«зафиксировать прямо сейчас»** — в коридоре клиники, сразу после получения результата анализа, когда только начались симптомы. Без мобайла пользователь отложит и забудет; данные окажутся неточными или потеряются.

Ключевой сценарий мобайла: **быстрый захват события** → структурированное хранение → просмотр истории в любой момент.

## Состав v1.0

### 1. Онбординг и авторизация

- Экран приветствия с объяснением модели (Проблема → Расследование → Вывод) — 3 слайда
- Регистрация (email + пароль) и вход
- Согласие на обработку медицинских данных

### 2. Дашборд — главный экран

- Список активных тематик с карточками
- На карточке: название, дата последнего события, количество событий
- FAB «+ Новая тематика»
- Пустое состояние с объясняющим текстом и CTA

### 3. Создание тематики

- Поле «Название проблемы» (обязательное)
- Поле «Описание» (необязательное)
- После создания — сразу переход внутрь тематики

### 4. Экран тематики — 3 вкладки

**Timeline (основная)**
- Хронологическая лента событий
- Карточки: иконка типа + дата + заголовок + краткое содержание
- FAB «+ Событие» → выбор типа

**Гипотезы**
- Список гипотез со статусом: `новая` / `проверяется` / `подтверждена` / `отклонена`
- Добавление: название + описание
- Смена статуса тапом

**Симптомы**
- Список симптомов проблемы
- Добавление из предустановленного списка или свободным текстом

### 5. Добавление события — 4 типа

**Консультация специалиста**
- Дата приёма
- Специальность / имя врача
- Диагноз (свободный текст)
- Назначения (свободный текст)
- Рекомендации (свободный текст)

**Анализ — ручной ввод**
- Название анализа, дата, лаборатория
- Построчный ввод параметров: название → значение → единица → мин норма → макс норма
- Автоматическая индикация: в норме / ниже / выше (зелёный / красный / жёлтый)

**Исследование (МРТ, КТ, УЗИ)**
- Тип исследования, дата
- Текстовое заключение
- Прикрепление фото документа (камера)

**Другое**
- Свободная форма: название + описание + дата

### 6. Карточка события (детальный просмотр)

- Все введённые данные
- Резолюция: «Помогло» / «Не помогло» + комментарий
- Привязка к гипотезам: `подтверждает` / `опровергает` / `нейтрально`

### 7. Профиль

- Имя, пол, дата рождения
- Выход из аккаунта

## Что НЕ входит в v1.0

| Функция | Причина |
|---|---|
| OCR / распознавание фото анализов | Требует AI pipeline |
| Графики динамики показателей | Нужно накопить данные |
| AI-ассистент (чат) | Требует LLM + RAG |
| Push-уведомления и напоминания | Post-MVP |
| Загрузка PDF | MinIO + обработка файлов |
| Курсы лечения и препараты | Достаточно текста в консультации |
| Авторизация через соцсети | Не критично для старта |
| Поиск и фильтрация | Нужна база событий для ценности |

## Технологический стек

**React Native + Expo** — выбор обоснован совместимостью с текущим стеком (TypeScript, Next.js):

| Компонент | Решение |
|---|---|
| Фреймворк | React Native + Expo SDK 51 |
| Навигация | Expo Router (file-based, аналог Next.js App Router) |
| Стейт / кэш | TanStack Query (react-query) |
| Формы | React Hook Form + Zod |
| UI Kit | NativeWind (Tailwind для RN) |
| Хранилище токена | expo-secure-store |
| HTTP | axios |
| Фото | expo-image-picker |
| Сборка | EAS Build (Expo Application Services) |

Бэкенд — тот же FastAPI. Новых эндпоинтов почти не требуется; мобайл использует те же `/api/v1/themes`, `/api/v1/events` и т.д.

### Экранный флоу

```
Splash
  ↓
[нет токена] → Онбординг → Регистрация / Вход
[есть токен] ↓
Dashboard
  ├── [тематика] → Экран тематики
  │     ├── Timeline → [событие] → Детальный просмотр → Резолюция
  │     │              [FAB +] → Выбор типа → Форма события
  │     ├── Гипотезы → [+] → Форма гипотезы
  │     └── Симптомы → [+] → Добавить симптом
  ├── [FAB +] → Создание тематики
  └── Профиль
```

## Отличия мобайла от веба

| Аспект | Веб | Мобайл v1 |
|---|---|---|
| Основной сценарий | Аналитика, просмотр истории | Быстрый захват события |
| Ввод анализов | Полная форма с таблицей | Построчный мобильный ввод |
| Фото документов | Загрузка файла | Камера (expo-image-picker) |
| Навигация | Сайдбар + вкладки | Bottom tab bar + stack |
| Дашборд | AI-инсайты, сводка | Список тематик, последние события |

## План разработки — 8 недель

### Неделя 1 — Инфраструктура и авторизация

- Инициализация Expo проекта, настройка EAS Build
- Expo Router: структура навигации (auth / app группы)
- Экраны регистрации и входа, JWT через expo-secure-store
- API клиент (axios + TanStack Query), базовая обработка ошибок
- Сплэш + онбординг (3 слайда)

### Неделя 2 — Тематики

- Dashboard: список тематик, пустое состояние, pull-to-refresh
- Создание тематики: форма, валидация, переход после создания
- Карточка тематики: статус, дата, счётчик событий
- Удаление тематики (свайп)

### Неделя 3 — Timeline и базовые события

- Экран тематики с тремя вкладками
- Timeline: хронологический список карточек событий
- Форма «Другое» (шаблон для остальных форм)
- FAB + bottom sheet выбора типа события

### Неделя 4 — Формы событий (консультация и исследование)

- Форма консультации: врач, диагноз, назначения, рекомендации
- Форма исследования: тип, заключение, прикрепление фото
- Детальный просмотр события
- Редактирование события

### Неделя 5 — Анализы

- Форма анализа: название, лаборатория, дата
- Построчный ввод параметров: значение / единица / норма мин / норма макс
- Цветовая индикация отклонений
- Просмотр параметров анализа

### Неделя 6 — Гипотезы и резолюции

- Список гипотез с визуальными статусами
- Создание гипотезы, смена статуса
- Резолюция события: «Помогло» / «Не помогло» + комментарий
- Привязка события к гипотезам (supports / rejects / neutral)

### Неделя 7 — Симптомы, профиль, polish

- Добавление симптомов к тематике
- Профиль: базовые данные, выход
- Пустые состояния для всех экранов
- Обработка ошибок сети, offline-состояние
- Haptic feedback, анимации переходов

### Неделя 8 — QA и публикация

- Полный прогон по всем сценариям на iOS и Android
- Фикс критических багов
- Скриншоты и описание для App Store / Google Play
- EAS Submit — отправка на ревью
- TestFlight (iOS) и Internal Track (Android)

## v2 — следующий релиз

1. **Динамика показателей** — графики изменения параметров анализов по времени
2. **OCR** — фото анализа → автоматическое распознавание параметров с верификацией пользователем
3. **Push-уведомления** — напоминания о повторном анализе, следующем приёме
4. **AI-инсайты** — интерпретация отклонений в анализах на основе истории
5. **Экспорт PDF** — отчёт по тематике для передачи врачу
