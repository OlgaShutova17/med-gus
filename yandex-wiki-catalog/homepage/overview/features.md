---
title: Функциональные модули
---

Карта функциональности по модулям с разбивкой на слои. Детальные use cases — в папке `use-cases/`.

---

## 1. Пользовательский модуль

**Регистрация и авторизация**
- Фронт: формы регистрации/авторизации, восстановление пароля
- Бэк: JWT-аутентификация, хеширование паролей
- БД: `users`

**Профиль пользователя**
- Фронт: личный кабинет, настройки профиля
- Бэк: API управления данными профиля
- БД: `users`, `user_profiles`

---

## 2. Модуль тематик

**Создание и управление тематиками**
- Фронт: список тематик, создание/редактирование
- Бэк: CRUD API для тематик
- БД: `themes`

**Привязка событий к тематикам**
- Фронт: выбор тематики при создании события
- Бэк: API связей событие↔тематика
- БД: `events.theme_id`

---

## 3. Модуль событий

**Консультации**
- Фронт: форма добавления консультации (файл или ручной ввод)
- Бэк: API консультаций
- БД: `events`, `consultations`

**Анализы**
- Фронт: загрузка файла (OCR) или ручной ввод показателей
- Бэк: API анализов, OCR-пайплайн
- БД: `events`, `analyses`, `analysis_results`

**Исследования**
- Фронт: форма с текстовым заключением и загрузкой файла
- Бэк: API исследований
- БД: `events`, `research`

---

## 4. Модуль гипотез

**Создание и ведение гипотез**
- Фронт: список гипотез внутри тематики, статусы
- Бэк: CRUD API гипотез
- БД: `hypotheses`

**Связь событий с гипотезами**
- Фронт: отметка «поддерживает / опровергает» на карточке события
- Бэк: API обновления связи
- БД: `event_hypotheses` (impact: supports | rejects | neutral)

---

## 5. Модуль напоминаний (Post-MVP)

**Система уведомлений**
- Бэк: сервис отправки push/email/Telegram
- Push при: напоминании об анализе, следующем приёме препарата

**Напоминания о курсе лечения**
- Фронт: расписание приёма в карточке курса
- Бэк: планировщик задач
- БД: `treatment_courses`, `medications`

---

## 6. Модуль аналитики

**Динамика показателей**
- Фронт: графики с цветовой индикацией нормы/отклонения
- Бэк: агрегация значений по параметру и периоду
- БД: `analysis_results` → `GET /analysis-results/dynamics`

**Статистика по резолюциям**
- Фронт: фильтрация событий по резолюции
- Бэк: агрегация помогло/не помогло
- БД: `resolutions`

---

## 7. Модуль AI-компонента

**Обработка анализов (OCR + NLP)**
- Бэк: OCR → Med NLP → извлечение параметров → верификация пользователем
- БД: `analysis_results`, `analysis_values`

**AI-инсайты и рекомендации**
- Бэк: анализ отклонений + knowledge graph → рекомендации специалиста
- БД: `ai_insights`, `ai_recommendations`

**AI-ассистент (чат)**
- Фронт: чат-интерфейс
- Бэк: RAG + knowledge graph + LLM
- API: `POST /ai/chat`

---

## 8. Модуль файлов

**Загрузка и хранение документов**
- Фронт: загрузка через форму или drag & drop
- Бэк: сохранение в MinIO/S3, ссылка в БД
- БД: `files`

---

## Общие компоненты

| Компонент | Описание |
|---|---|
| Контроль доступа | Row-level security: пользователь видит только свои данные |
| Поиск и фильтрация | Поиск по ФИО врача, диагнозу; фильтрация по типу события и резолюции |
| Timeline | Единая хронологическая лента всех событий по тематике |
