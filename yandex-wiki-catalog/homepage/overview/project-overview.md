## Описание продукта

Веб-приложение (и/или Telegram-бот), которое помогает пользователям вести проблемно-ориентированный медицинский дневник: отслеживать состояние здоровья, динамику заболеваний, назначенные препараты, результаты анализов и взаимодействие со специалистами.

Встроенный AI-агент формирует гипотезы причин проблемы и даёт рекомендации, опираясь на симптомы, историю анализов и записи консультаций.

---

## Проблема (AS IS)

- Лечение одной и той же проблемы «расфокусировано» по разным источникам: почта, приложения клиник, печатные документы
- Нет возможности видеть динамику по конкретному показателю анализа
- Когда проблема длится долго — сложно вспомнить, какие гипотезы уже проверялись, к каким специалистам обращались, какие препараты помогли
- Существующие приложения клиник привязаны к конкретной клинике и дают документооборот, а не проблемно-ориентированное представление

## Цель (TO BE)

Создать приложение, которое позволит:

- фиксировать проблему / симптом
- формировать гипотезы причин с помощью AI-агента
- фиксировать обращения к специалистам в разрезе проблем
- сохранять назначения, анализы, препараты, симптомы
- отмечать результат: «помогло» / «не помогло»
- видеть динамику показателей анализов

---

## Функциональные требования

1. Система должна предоставлять возможность создавать и сохранять отдельные записи по каждой проблеме/тематике.
2. Система должна обеспечивать ведение хронологической истории действий по каждой проблеме.
3. Система должна позволять фиксировать гипотезы относительно причин возникновения проблемы.
4. Система должна позволять указывать способы проверки гипотезы (поход к врачу / упражнения / препараты).
5. Система должна предоставлять возможность фиксировать результат проверки гипотезы (подтвердилась или нет).
6. Система должна позволять регистрировать обращения к врачам и специалистам в рамках конкретной проблемы.
7. Система должна обеспечивать хранение назначений, полученных от специалиста, в привязке к проблеме.
8. Система должна предоставлять возможность фиксировать проведённые медицинские исследования и анализы.
9. Система должна позволять вручную вводить числовые значения медицинских показателей.
10. Система должна предоставлять информацию о динамике изменения конкретного медицинского показателя за весь период.
11. Система должна обеспечивать поиск и просмотр всех назначений и исследований по выбранной проблеме (по ФИО врача, диагнозу).
12. Система должна обеспечивать разграничение данных по пользователям: каждый пользователь имеет доступ только к своим данным.
13. Система должна предоставлять возможность взаимодействовать с AI-агентом для получения рекомендаций по проверке гипотез (режим диалога).
14. Система должна использовать AI-компонент для помощи в структурировании данных при вводе результатов анализов.

---

## Нефункциональные требования

1. **Конфиденциальность** — система обеспечивает конфиденциальность медицинских данных в соответствии с законодательством о защите персональных данных.
2. **Изоляция данных** — пользователи не имеют технической возможности получать доступ к данным других пользователей (row-level security).
3. **Согласие на обработку данных** — при регистрации предусмотрен механизм получения согласия на обработку персональных и медицинских данных.
4. **Производительность** — время отклика при просмотре истории проблемы и получении статистики по показателям не более 1 секунды.
5. **Простота интерфейса** — минималистичный UI, не требующий сложного обучения.

## Фичи на будущее (Post-MVP)

- **OCR** — распознавание текста из фото или PDF для автоматического извлечения показателей анализов
- **Напоминания** — уведомления о необходимости сдать повторный анализ
- **Автоматические алерты** — уведомление при выходе показателя за пределы нормы
- **Фильтрация по резолюции** — «помогло» / «не помогло»
- **Фильтрация по типу события** — анализы, консультации, исследования, другое
- **Поделиться результатом** — отправить событие с резолюцией другому пользователю или врачу
- **AI-диагностика** — автоматическое формирование гипотез на основе паттернов данных
- **Сообщество пациентов** — анонимный обмен опытом лечения схожих проблем
- **Интеграции с лабораториями** — автоматический импорт результатов анализов
