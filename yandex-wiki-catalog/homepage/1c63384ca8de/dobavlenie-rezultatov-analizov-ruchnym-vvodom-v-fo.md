## **User Story**

Как пользователь, я хочу вручную вводить результаты анализов в приложении, чтобы хранить их в одном месте.

## **Описание**

Пользователь получает результаты медицинских анализов не только в электронном виде, но и на бумажных бланках или в системах, откуда нельзя скачать файл. Он хочет всё равно зафиксировать эти результаты в приложении. В приложении пользователь заходит в раздел «Анализы», создаёт новую запись анализа, указывает его тип (например, кровь, моча) и дату проведения, а затем вручную вводит ключевые показатели по выданной форме. После сохранения анализ добавляется в общий список анализов пользователя и остаётся доступным при последующих входах.

## **Бизнес правила**

1. Доступ только авторизованному пользователю
   Сохранять результаты анализов может только пользователь, вошедший в приложение под своей учётной записью.

2. Обязательные данные для сохранения анализа (ручной ввод)
   Анализ может быть сохранён только при наличии:

   * типа анализа;

   * даты проведения;

   * хотя бы одного заполненного поля с результатом (например, значения, заключение, комментарий).

3. Допустимые типы анализов
   Тип анализа выбирается пользователем из преднастроенного списка (общий анализ крови, биохимия, анализ мочи, и т.д.); при необходимости пользователь может выбрать «другое» и указать свой вариант.

4. Структура данных результатов
   В зависимости от типа анализа приложение может предлагать:

   * либо структурированный ввод (название показателя, значение, единицы, референсные диапазоны, пометка «норма/не норма»),

   * либо общее текстовое поле для результатов/заключения.

5. Привязка анализа к профилю
   Каждый вручную введённый анализ привязан к конкретному профилю пользователя и отображается только ему.

6. Пользователь вводит только числа с дробными и целые, да/нет.

## **Acceptance Criteria**

1. Пользователь, будучи авторизованным, может открыть раздел «Анализы» и экран добавления нового анализа с режимом ручного ввода.

2. Пользователь может выбрать тип анализа из списка (биохимия, общий анализ крови и т.д.).

3. Пользователь может указать дату проведения анализа.

4. Пользователь может вручную ввести результаты анализа:

5. При заполненных обязательных полях (тип, дата и хотя бы одно поле с результатами) и нажатии «Сохранить» создаётся запись анализа и отображается в общем списке анализов пользователя.

6. После повторного входа в приложение вручную введённые анализы доступны пользователю в общем списке и при открытии показывают введённые значения/текст.

7. При попытке сохранения без типа анализа, даты или без каких‑либо данных результата система не создаёт запись и показывает сообщение об ошибке с указанием, что нужно заполнить.

## **Use Case: Добавление анализа ручным вводом**

* Название: UC‑2 «Добавление результата анализа вручную»

* Актор: авторизованный пользователь, система

* Предусловия:

  * Пользователь зарегистрирован и вошёл в приложение.

  * У пользователя есть результат анализа, который он может прочитать (с бумажного бланка, портала и т.д.), но нет файла для прикрепления.

## **Основной поток**

 1. Пользователь открывает приложение и переходит в раздел «Анализы».

 2. Пользователь нажимает кнопку «Добавить анализ вручную».​

 3. Система отображает форму ручного ввода анализа: тип анализа, дата проведения, поля для результатов/заключения и, при необходимости, комментарий.

 4. Пользователь выбирает тип анализа из списка.

 5. Пользователь указывает дату проведения анализа.

 6. Пользователь вручную вводит данные результата.

 7. Пользователь нажимает кнопку «Сохранить».

 8. Система проверяет, что обязательные поля заполнены корректно.

 9. Система сохраняет запись анализа и привязывает её к профилю пользователя.

10. Система показывает уведомление об успешном сохранении и обновлённый список анализов с новой записью.

## **Альтернативные потоки**

**A1. Не заполнены обязательные поля**

* На шаге 7 пользователь нажимает «Сохранить», не указав тип анализа, дату или не введя никакого результата.

* Система выполняет валидацию, подсвечивает проблемные поля и показывает сообщения об ошибке (например, «Выберите тип анализа», «Укажите дату», «Введите хотя бы один результат»).

* Пользователь исправляет данные и повторяет шаг 7.

---

## **Given–When–Then**

1. Успешное добавление анализа вручную

   * Given авторизованный пользователь находится в режиме ручного ввода анализа

   * When он выбирает тип анализа, указывает дату проведения и вводит результаты (значения или заключение), затем нажимает «Сохранить»

   * Then система создаёт запись анализа и показывает её в общем списке анализов пользователя.

2. Валидация обязательных полей при ручном вводе

   * Given авторизованный пользователь находится в режиме ручного ввода анализа

   * When он нажимает «Сохранить», не указав тип анализа, дату или не введя никакого результата

   * Then система не создаёт запись анализа и показывает сообщение о необходимости заполнить обязательные поля.
