title: Загрузка результатов анализов прикрепляемым файлом

Как пользователь, я хочу загружать результаты анализов в приложение, чтобы хранить их в одном месте.

## **Описание**

Пользователь получает результаты медицинских анализов в разных форматах (электронные PDF от лабораторий, сканы или фото бумажных бланков) и хочет перестать хранить их в мессенджерах, почте и галерее телефона.
В приложении он заходит в раздел «Анализы», создаёт новый запись анализа, указывает его тип (например, кровь, моча) и дату проведения, а затем прикрепляет файл результата.
После сохранения анализ добавляется в общий список анализов пользователя и остаётся доступным при последующих входах в приложение.

### **Бизнес  правила**

1. Доступ только авторизованному пользователю
   Сохранять результаты анализов может только пользователь, вошедший в приложение под своей учётной записью; неавторизованный пользователь не видит форму добавления анализа.

2. Обязательные данные для сохранения анализа
   Анализ может быть сохранён только при наличии всех обязательных данных: типа анализа, даты проведения и хотя бы одного файла результата.

3. Допустимые типы анализов
   Тип анализа выбирается пользователем из преднастроенного списка (например, общий анализ крови, биохимия, анализ мочи, МРТ и т.д.); произвольный ввод возможен только если это явно предусмотрено продуктом (опция «другое»).

4. Ограничение по дате анализа
   Дата проведения анализа не может быть пустой и не может быть позже текущей даты .

5. Допустимые форматы файлов результата
   В качестве файла результата могут использоваться только поддерживаемые форматы ( PDF и PNG. JPG, JPEG); файлы других типов не принимаются системой.

6. Привязка анализа к профилю пользователя
   Каждый сохранённый анализ однозначно привязан к конкретному профилю пользователя и не виден другим пользователям
   7\. Размер. файла

### **Acceptance Criteria**

1. Пользователь может заполнить форму по  выбраному типу анализа ( биохимия, общий анализ крови, и т.д.)
2. Пользователь может загрузить файл с результатами анализа (PDF и PNG. JPG, JPEG ) размером не больше 25мб.
3. Пользователь может указать тип анализа (например, кровь, моча и т.д.) и дату проведения.
4. Пользователь может просмотреть содержимое анализа.
5. Все данные сохраняются в профиле пользователя и доступны при следующем входе в приложение.

## **Use Case: Добавление нового анализа** прикрепляемым файлом

* Название: UC‑1 «Добавление результата анализа»

* Актор: авторизованный пользователь, система

* Предусловия:

  * Пользователь зарегистрирован и вошёл в приложение.

  * У пользователя есть результат анализа в виде файла (PDF или фото).

## **Основной сценарий**

 1. Пользователь открывает приложение и переходит в раздел «Анализы».

 2. Пользователь выбирает режим добавления анализа с прикреплением файла результата.

 3. Система отображает форму добавления анализа с полями: тип анализа, дата проведения, файл результата (и необязательные поля, если есть).

 4. Пользователь выбирает тип анализа из списка (например, биохимия, общий анализ крови и т.д.).

 5. Пользователь указывает дату проведения анализа.

 6. Пользователь прикрепляет файл результата анализа (PDF или изображение).

 7. Пользователь нажимает кнопку «Сохранить».

 8. Система проверяет, что все обязательные поля заполнены корректно.

 9. Система сохраняет запись анализа, привязывая её к профилю пользователя.

10. Система показывает уведомление об успешном сохранении и обновлённый список анализов, где отображается новая запись.

## **Альтернативные сценарии**

**A1. Не заполнены обязательные поля**

* На шаге 7 пользователь нажимает «Сохранить», не указав тип анализа, дату или не прикрепив файл.

* Система выполняет валидацию и обнаруживает незаполненные/некорректные поля.

* Система не сохраняет анализ и подсвечивает проблемные поля, показывая сообщения об ошибке (например, «Выберите тип анализа», «Укажите дату», «Прикрепите файл»).

* Пользователь исправляет данные и повторяет шаг 7.

**A2. Неподдерживаемый формат файла**

* На шаге 6 пользователь выбирает файл неподдерживаемого формата.

* Система отклоняет файл и показывает сообщение о допустимых форматах (PDF, изображение).

* Пользователь выбирает файл в поддерживаемом формате и повторяет шаг 6.

## **Постусловия**

* В профиле пользователя появляется новая запись анализа с указанными типом, датой и прикреплённым файлом.

* Пользователь видит эту запись в общем списке анализов и может открыть её для просмотра.

---

## **Given–When–Then**

1. Успешное добавление анализа

   * Given авторизованный пользователь находится на экране добавления анализа;

   * When он выбирает тип анализа, указывает дату проведения и прикрепляет файл результата, а затем нажимает «Сохранить»;

   * Then система создаёт запись анализа и показывает её в общем списке анализов пользователя.

2. Валидация обязательных полей

   * Given авторизованный пользователь находится на экране добавления анализа;

   * When он нажимает «Сохранить», не указав тип анализа, дату или не прикрепив файл;

   * Then система не создаёт запись анализа и показывает сообщение о необходимости заполнить обязательные поля.

3. Доступность после повторного входа

   * Given у пользователя ранее сохранены результаты анализов в приложении;

   * When он выходит из аккаунта, затем снова входит под той же учётной записью и открывает раздел «Анализы»;

   * Then система отображает ранее сохранённые анализы в общем списке.
