### **Точка входа и выбор метода**

1. Пользователь нажимает кнопку «Добавить анализ». Система предлагает два пути:
   •  Путь А: Загрузить документ (Фото/PDF).
   •  Путь Б: Ввести данные вручную.

---

### **Путь А: Загрузка файла (Автоматизированный поток)**

1. Загрузка и предварительная обработка: Пользователь загружает файл. Система проверяет формат и качество (если это фото — достаточно ли оно четкое, понятное).
2. OCR \+ Med-NLP (Распознавание):
   •  Использование OCR (Optical Character Recognition) для перевода картинки в текст.
   •  Использование специализированной языковой модели (LLM), обученной на медицинских терминах, для извлечения сущностей:
   Параметр\* (например, Гемоглобин).
   Значение\* (например, 140).
   Единицы измерения\* (г/л).
   Референсные значения\* (120–160).
3. Верификация пользователем (Обязательный этап):
   •  Система выводит распознанные данные в виде таблицы.
   •  Важно: Пользователь должен подтвердить корректность. Если ИИ ошибся в цифре, пользователь правит её вручную. Это снимает часть юридических рисков с приложения.

---

### Путь Б: Ручной ввод (Структурированный поток)

1. Поиск параметра: Пользователь начинает вводить название (например, «Глюкоза»). Система предлагает автодополнение из справочника (МКБ или локальной базы лабораторий).
2. Заполнение полей:
   •  Значение (числовое поле).
   •  Единицы измерения (выпадающий список, подтянутый под конкретный параметр).
   •  Референсы (подсказываются автоматически, если они есть в базе, но доступны для правки).
3. Привязка к контексту: Пользователь может указать дату сдачи и лабораторию.

---

### Работа ИИ-анализатора (Core Logic)

*После сохранения данных (из любого пути) включается ИИ-модуль анализа:*

1. Сравнение с нормой.
   Система подсвечивает отклонения (цветовая индикация: красный — критично, желтый — погранично, зеленый — норма).
2. Контекстуальный анализ.
   ИИ анализирует не один параметр в вакууме, а совокупность:
   •  Пример: Высокий сахар \+ Высокий индекс HOMA \+ Жалобы пользователя на жажду.
   •  Учет профиля: Пол, возраст, хронические заболевания из анкеты пользователя.
3. Формирование «Второго мнения» (Insights).
   ИИ генерирует текст по шаблону:
   •  Summary: «Ваши показатели \[X\] выше нормы, что может свидетельствовать о \[процессе Y\]».
   •  Предупреждение: «Данная информация носит ознакомительный характер и не является диагнозом».
4. Рекомендация маршрутизации.
   Алгоритм сопоставляет отклонения со специализациями врачей:
   •  Если повышен ТТГ -\> Рекомендация: Эндокринолог.
   •  Если низкий ферритин и железо -\> Рекомендация: Терапевт / Гематолог.
   •  Если есть резолюция пользователя «не помог предыдущий прием» -\> ИИ предлагает рассмотреть мнение врача другого профиля или более узкого специалиста.
