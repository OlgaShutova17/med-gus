## User Story

Как  пользователь, я хочу добавлять результаты исследований в приложении, чтобы хранить их в одном месте.

### Описание

Пользователь проходит инструментальные исследования (МРТ, КТ, УЗИ, ЭКГ и т.п.) в разных клиниках и получает результаты в виде заключений, протоколов, изображений или файлов, которые теряются в мессенджерах, почте и на дисках. Он хочет фиксировать результаты всех исследований в одном приложении: заходить в раздел «Исследования», создавать новую запись, указывать тип исследования (например, МРТ коленного сустава, КТ лёгких, УЗИ брюшной полости), дату проведения, а также вводить заключение и при необходимости прикреплять файлы. После сохранения исследование добавляется в общий список исследований пользователя и хранится на его аккаунте.

## **Бизнес правила**

1. Доступ только авторизованному пользователю
   Сохранять результаты исследований может только пользователь, вошедший в приложение под своей учётной записью; неавторизованный пользователь не видит форму добавления исследования.

2. Обязательные данные для сохранения исследования
   Исследование может быть сохранено только при наличии:

   * типа исследования;

   * даты проведения;

   * хотя бы одного заполненного поля с результатом или прикреплённого файла с результатом.

3. Допустимые типы исследований
   Тип исследования выбирается пользователем из преднастроенного списка (МРТ, КТ, УЗИ, ЭКГ и т.д., с уточняющими вариантами, например «МРТ головного мозга», «УЗИ щитовидной железы»); при необходимости пользователь может выбрать «другое» и указать свой вариант.

4. Структура данных результатов
   В зависимости от типа исследования приложение может предлагать:

   * либо структурированный ввод (область исследования, протокол, ключевые находки, рекомендации);

   * либо общее текстовое поле для результатов/заключения врача и комментариев пользователя.

5. Работа с файлами исследования
   В качестве файлов результата могут использоваться только поддерживаемые форматы (PDF, PNG/JPG ); файлы неподдерживаемых форматов или превышающие максимально допустимый размер (25мб) не принимаются системой.

6. Ограничение по дате исследования
   Дата проведения исследования не может быть пустой и не может быть позже текущей даты.

7. Привязка исследования к профилю
   Каждое сохранённое исследование однозначно привязано к конкретному профилю пользователя и не отображается другим пользователям.

8. Формат вводимых данных
   В структурированных полях пользователь вводит только корректные значения по типу поля (числовые поля — целые/дробные числа, булевые — да/нет), а в текстовых полях — свободный текст заключения и комментариев.

## **Acceptance Criteria**

1. Пользователь, будучи авторизованным, может открыть раздел «Исследования» и экран добавления нового исследования.

2. Пользователь может выбрать тип исследования из преднастроенного списка (МРТ, КТ, УЗИ и т.д., включая «другое» при необходимости).

3. Пользователь может указать дату проведения исследования, и система не допускает дату позже текущей.

4. Пользователь может ввести результаты исследования: заполнить структурированные поля (при наличии) и/или общее текстовое поле с заключением и комментариями.

5. Пользователь может прикрепить файл с результатом исследования в допустимом формате и размере.

6. При заполненных обязательных полях (тип, дата и хотя бы одно поле с результатами или прикреплённый файл) и нажатии «Сохранить» создаётся запись исследования и отображается в общем списке исследований пользователя.

7. После повторного входа в приложение сохранённые исследования доступны пользователю в общем списке и при открытии показывают введённые данные и/или прикреплённые файлы.

8. При попытке сохранения без типа исследования, даты или без каких‑либо данных результата система не создаёт запись и показывает сообщение об ошибке с указанием, что нужно заполнить.

## **Use Case: Добавление результата исследования**

* Название: UC‑3 «Добавление результата исследования»

* Актор: авторизованный пользователь, система

* Предусловия:

  * Пользователь зарегистрирован и вошёл в приложение.

  * У пользователя есть результат исследования (заключение, протокол, снимки), который он может прочитать или загрузить.

## **Основной поток**

 1. Пользователь открывает приложение и переходит в раздел «Исследования».

 2. Пользователь нажимает кнопку «Добавить исследование».

 3. Система отображает форму добавления исследования: тип исследования, дата проведения, поля для результата/заключения и, при необходимости, комментарий и прикрепление файлов.

 4. Пользователь выбирает тип исследования из списка.

 5. Пользователь указывает дату проведения исследования.

 6. Пользователь вводит данные результата (заключение, ключевые находки, рекомендации) и при необходимости прикрепляет файл результата в поддерживаемом формате.

 7. Пользователь нажимает кнопку «Сохранить».

 8. Система проверяет, что обязательные поля заполнены корректно и, при наличии файла, формат и размер допустимы.

 9. Система сохраняет запись исследования и привязывает её к профилю пользователя.

10. Система показывает уведомление об успешном сохранении и обновлённый список исследований с новой записью.

## **Альтернативные потоки**

**A1. Не заполнены обязательные поля**

* На шаге 7 пользователь нажимает «Сохранить», не указав тип исследования, дату или не введя никакого результата.

* Система выполняет валидацию, подсвечивает проблемные поля и показывает сообщения об ошибке (например, «Выберите тип исследования», «Укажите дату», «Введите хотя бы один результат»).

* Пользователь исправляет данные и повторяет шаг 7.

**A2. Неподдерживаемый формат файла или превышен размер**

* Пользователь выбирает файл неподдерживаемого формата или превышающий максимально допустимый размер.

* Система отклоняет файл и показывает сообщение о допустимых форматах и размере.

* Пользователь выбирает файл в поддерживаемом формате и допустимого размера и повторяет попытку.

## **Given–When–Then**

1. Успешное добавление исследования

   * Given авторизованный пользователь находится на экране добавления исследования

   * When он выбирает тип исследования, указывает дату проведения и вводит результаты (и при необходимости прикрепляет файл), затем нажимает «Сохранить»

   * Then система создаёт запись исследования и показывает её в общем списке исследований пользователя.

2. Валидация обязательных полей при добавлении исследования

   * Given авторизованный пользователь находится на экране добавления исследования

   * When он нажимает «Сохранить», не указав тип исследования, дату или не введя никакого результата

   * Then система не создаёт запись исследования и показывает сообщение о необходимости заполнить обязательные поля.

3. Доступность после повторного входа

   * Given у пользователя ранее сохранены результаты исследований в приложении

   * When он выходит из аккаунта, затем снова входит под той же учётной записью и открывает раздел «Исследования»

   * Then система отображает ранее сохранённые исследования в общем списке.
