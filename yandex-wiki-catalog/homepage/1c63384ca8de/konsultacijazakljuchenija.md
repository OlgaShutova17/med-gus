Как пациент, я хочу видеть результаты консультации внутри системы, чтобы понимать причинно-следственные связи своего заболевания и иметь возможность стабилизировать состояние.

### **Критерии приемки**

1. Наличие пройденной консультации специалиста в напечатанном виде.

2. Возможность добавления данных о консультации в ручном режиме, наличие полей для ввода данных консультации.

3. Возможность добавления материалов (лист приема) путем загрузки файла в формате jpg/jpeg/pdf.

4. Кнопка "Добавить консультацию" кликабельна.

5. Нажатие на кнопку "Добавить консультацию" открывает модельное окно с возможностью выбрать файл или перетащить файл путем "Drag&Drop".

6. Возможность добавления заметки пользователя по итогу консультации.

7. Кнопка "Сохранить" кликабельна.

8. Возможность сохранить данные.

### **Use Case:**

#### Добавление консультации.

Приоритет:
Высокий.

Акторы:
Пациент, система.

Предусловия:
1\. Пользователь авторизован в системе.
2\. Пользователь посетил прием врача.

### Основной сценарий:

 1. Пользователь находится на главной странице.

 2. Заходит в раздел "Мои консультации".

 3. Пользователь нажимает "Добавить консультацию".

 4. Пользователь видит открытое модальное окно "Добавить консультацию" с областью загрузки и кнопкой "Обзор", "Добавить вручную", "Сохранить".

 5. Пользователь просматривает список файлов, доступный через интерфейс компьютера путем нажатия на кнопку "Обзор".

 6. Пользователь выделяет нужный файл.

 7. Пользователь нажимает "Выбрать".

 8. Пользователь нажимает кнопку "Сохранить" в модальном окне "Добавить консультацию".

 9. Форма сохраняется, отображается в интерфейсе.

10. Загруженная форма сохраняется в базе данных.

### Альтернативные сценарии:

**А1. Пользователь перетаскивает файл-скан с консультацией, используя мышь.**

 1. Пользователь находится на главной странице.

 2. Заходит в раздел "Мои консультации".

 3. Пользователь нажимает "Добавить консультацию".

 4. Пользователь видит открытое модальное окно "Добавить консультацию" с областью загрузки и кнопкой "Обзор", "Добавить вручную", "Сохранить".

 5. Пользователь просматривает список файлов, доступный через интерфейс компьютера, открыв нужную папку в системе.

 6. Пользователь выделяет нужный файл.

 7. Пользователь перемещает файл в область загрузки путем перетаскивания с помощью мыши, удерживая нажатой левую кнопку мыши.

 8. Пользователь нажимает кнопку "Сохранить" в модальном окне "Добавить консультацию".

 9. Форма сохраняется, отображается в интерфейсе.

10. Загруженная форма сохраняется в базе данных.

**А2.** **Пользователь добавляет данные о консультации вручную.**

1. Пользователь находится на главной странице.

2. Пользователь нажимает "Добавить консультацию".

3. Пользователь видит открытое модальное окно "Добавить консультацию" с областью загрузки и кнопками "Обзор", "Добавить вручную", "Сохранить".

4. Пользователь нажимает кнопку "Добавить вручную".

5. Открывается окно "добавить консультации" в полноэкранном режиме, в котором есть соответствующие поля заполнения:
   1\. Клиника? 
   2\. Врач? 
   1\. Анамнез жизни.
   2\. Анамнез заболевания
   3\. Течение заболевания.
   4\. Диагноз.
   5\. Лечение.
   6\. Рекомендации.
   7\. Заключение.
   8\. Повторная явка.
   \* - все разделы являются необязательными к заполнению и могут сохраняться при пустом вводе.

6. Пользователь заполняет желаемые разделы.

7. Пользователь нажимает кнопку "Сохранить" в полноэкранном окне "Добавить консультацию".

8. Введенные данные сохраняются, отображается в интерфейсе.

9. Загруженные данные сохраняются в базе данных.
